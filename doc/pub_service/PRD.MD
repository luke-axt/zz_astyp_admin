# 公网服务选型

## 当前系统架构

内网系统1：技术栈：python+window+mysql/sqlite,包含如下功能：
1. 实时数据采集：通过rpa、api等方式从外部系统或者公司外购系统采集数据。
2. 离线数仓：以数据处理和指标计算为主，主要就是跑sql。
3. window桌面应用：基于pyside6的桌面应用，主要给用户下载报表，维护业务数据用的。
4. rpa模块：负责跨系统的重复性操作的自动化，或者支持erp无法实现的关键功能（比如现在的尾程，或者外贸业务订单同步等）。
5. 定时任务模块：所有数据采集、离线数仓、rpa等需要定时启动的任务，都由定时任务模块负责调度，主要是通过apschedule实现，目前已经实现的分布式运行、T+1跑批的事件依赖等。
6. 有些需要7*24小时处理的任务，我会独立部署在云服务器，跟我内网生产环境隔离开。（我内网是没有不间断电源的）

整个项目相当于一个桌面应用+大量批处理任务，批处理任务已经超过200个。系统已经对接8个海外仓系统、1个erp、1个金蝶云星空系统，还集成了百度云、腾讯云、163邮件、企业微信等基础服务能力。

目前我们是部署在window服务器，方便做爬虫。另外，有些需要7*24小时的任务，我们是做基于sqlite的单独服务部署在云服务器上，最大程度节约硬件成本。
